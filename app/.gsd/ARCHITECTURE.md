- **AdminLayout.jsx**: Proporciona el Sidebar y Topbar específicos para el equipo interno del SaaS.

- **ClientDashboard.jsx**: Resumen de actividad del cliente.

## Data Flow

1. **Authentication Flow**: `AuthProvider` (en `AuthContext.jsx`) envuelve la aplicación, proveyendo estado `user`, `login` y `logout`.
2. **Routing**: `App.jsx` actúa como el despachador central basado en rutas de `react-router-dom`.
3. **Layout Wrapper**: Las rutas anidadas inyectan contenido en los Layouts correspondientes mediante `<Outlet />` de React Router.
4. **State (Mock Data)**: La data de usuarios y módulos está actualmente simulada (Mock) para validación de UI/UX.

## Conventions

**Naming**: CamelCase para componentes (`AdminLayout.jsx`), kebab-case para assets.
**Structure**: Separación clara entre Layouts (shells) y Pages (feature areas). Contextos para estado global.
**Styling**: Uso exclusivo de Tailwind v4 y variables CSS nativas en `@theme`.
