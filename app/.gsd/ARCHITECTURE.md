## Overview

PKT ERP is a multi-tenant web application built with React and Vite. It provides a comprehensive suite of ERP modules (CRM, Inventory, Sales, Finance, Projects, Calendar) for clients (tenants), as well as a SuperAdmin dashboard for managing clients and global activity. Data persistence and authentication are handled by Firebase (Firestore and Firebase Auth).

## Components

### App Router
- **Purpose:** Handles main application routing and protects routes based on user role (SuperAdmin vs Client) and module permissions.
- **Location:** `src/App.jsx`
- **Dependencies:** `react-router-dom`, context providers, layout components.

### AuthContext
- **Purpose:** Manages authentication state, fetches user profiles, handles multi-tenancy context (`companyId`), and enforces role-based access.
- **Location:** `src/context/AuthContext.jsx`
- **Dependencies:** Firebase Auth, Firestore.

### Client Modules
- **Purpose:** Core business logic and UI for tenant-specific features (CRM, Inventory, Sales, Finance, Calendar, Projects).
- **Location:** `src/modules/client/*`
- **Dependencies:** React hooks (`useCrm`, `useSales`, etc.), Zod for validation, Lucide React for icons.

### Admin Modules
- **Purpose:** SuperAdmin interfaces for managing subscriptions, viewing activity logs, and managing all organizations.
- **Location:** `src/modules/admin/*`
- **Dependencies:** Custom admin hooks, Firebase Firestore.

## Data Flow

1. User authenticates via Firebase Auth. `AuthContext` retrieves user profile from Firestore `users` collection.
2. If the user belongs to an organization, the `companyId` is stored in the context to scope all subsequent queries.
3. Views and components invoke custom hooks (e.g., `useProjects`, `useInventory`).
4. Custom hooks execute scoped Firestore queries (e.g., fetching documents where `companyId` matches the tenant's ID).
5. State is managed locally in React, combined with real-time updates (snapshots) from Firestore.
