# Plan 18.1: Rediseño del Dashboard (Tarjetas Unificadas por Módulo)

## Objective
Simplificar y rediseñar el Dashboard de Cliente (`ClientDashboard.jsx`). En lugar de tener una sección separada de KPIs generales y otra de enlaces a módulos, se debe renderizar **UNA ÚNICA TARJETA por cada módulo** que el usuario tenga **activado** en `user?.subscription?.activeModules`. Cada tarjeta servirá como resumen (mostrando la información clave / KPIs de ese módulo) y como botón de acceso directo.

## Context
- `app/src/modules/client/dashboard/ClientDashboard.jsx`
- `app/src/context/AuthContext.jsx` -> Proveedor de `user` con `subscription.activeModules`.
- Objetivos de diseño: Mantener la estética premium y moderna (glassmorphism/dark tones) del sistema.

      3. Botón / Indicador de acción (Ej: "Entrar al módulo") para navegar a la ruta correspondiente (`/client/[module]`).
