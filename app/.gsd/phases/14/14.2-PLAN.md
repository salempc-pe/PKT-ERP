# Plan 14.2: Setup Password Public Route

## Objective
Construir y conectar la interfaz gráfica pública encargada de reclamar la invitación y asociar una contraseña al usuario nuevo, y conectar las ramas del `react-router-dom`.

## Context
- .gsd/SPEC.md
- src/App.jsx
- src/modules/SetupPassword.jsx

## Tasks

<task type="auto">
  <name>Create SetupPassword.jsx</name>
  <files>src/modules/SetupPassword.jsx</files>
  <action>
    - Crear un componente funcional público `SetupPassword`.
    - Usar el mismo estilo gráfico minimalista/glassmorphism de `Login.jsx` o similar para que el onboarding sea agradable.
    - Leer el token de la URL usando `useSearchParams()` o la query string `?token=XYZ`.
    - Disponer un formulario de Contraseña y Confirmar Contraseña (mínimo 4 caracteres).
    - Si ambas contraseñas coinciden, ejecutar `setupUserPassword(token, password)` (importado vía `useAuth`).
    - Al completarse con éxito, se podrá redirigir al login o incluso auto-login si se desea. Sin embargo, para no complicar el contexto, una vez logrado emitir un mensaje de éxito con un link (o simple `navigate('/')`) es suficiente.
  </action>
  <verify>test -f src/modules/SetupPassword.jsx</verify>
  <done>Vista pública terminada y que se comunique efectivamente con el contexto Auth.</done>
</task>

<task type="auto">
  <name>Connect Route to App</name>
  <files>src/App.jsx</files>
  <action>

    - Agregar una ruta libre (pública) `<Route path="/setup-password" element={<SetupPassword />} />`.
  </action>
  <verify>grep -q "\/setup-password" src/App.jsx</verify>

## Success Criteria
- [ ] Componente público existe de manera estéticamente alineada al diseño.
- [ ] Las validaciones obligan al usuario a confirmar su nueva contraseña con exactitud.
- [ ] El guardado activa la sesión e invita al usuario al app.
