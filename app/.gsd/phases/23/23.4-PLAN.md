# Plan 23.4: Validación de Límites de Cuotas (maxUsers)

## Objective
Implementar bloqueos en la creación de nuevos usuarios si se ha alcanzado la cuota de `maxUsers` definida para la organización.

## Context
- .gsd/ROADMAP.md
- src/context/AuthContext.jsx
- src/modules/client/team/TeamModule.jsx

## Tasks

<task type="auto">
  <name>Implementar validación de cuotas de usuario</name>
  <files>
    - src/context/AuthContext.jsx
    - src/modules/client/team/TeamModule.jsx
  </files>
  <action>
    En `AuthContext.jsx`:
    - Modificar la función `clientCreateUser` (o la función usada para invitar) y asegurar que antes de generar el token, se haga un chequeo:
      `const currentOrg = mockOrganizations.find(o => o.id === orgId);`
      `const activeUsers = mockUsers.filter(u => u.organizationId === orgId).length;`
      `const limit = currentOrg.maxUsers || 5;`
      Si `activeUsers >= limit`, retornar un error indicando que se alcanzó el límite contratado.
    
    En `TeamModule.jsx`:
    - Manejar este error e indicárselo al usuario mediante un alert o UI interactiva: "Has alcanzado el límite de {maxUsers} usuarios. Contacta a soporte para subir de plan."
    - Deshabilitar el botón de invitar si el número actual de usuarios ya alcanzó el límite.
  </action>
  <verify>grep "maxUsers" src/context/AuthContext.jsx && grep "límite" src/modules/client/team/TeamModule.jsx</verify>
  <done>No es posible crear más usuarios que los permitidos por el campo maxUsers de la organización.</done>
</task>

## Success Criteria
- [ ] El sistema bloquea creación de usuarios (en el backend simulado/auth context) cuando se alcanza el límite.
- [ ] El módulo de equipo indica visualmente la cuota de uso (ej. 3/5 usuarios registrados) y deshabilita el ingreso.
