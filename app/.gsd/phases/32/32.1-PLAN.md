# Plan 32.1: Implementación del Módulo de Terrenos Inmobiliarios

Este plan detalla la creación del módulo de base de datos de terrenos con vista de tabla, filtros, calculadora reactiva y pipeline comercial (Kanban).

## Contexto
Referencia: ROADMAP.md — Fase 32

## Arquitectura y Capa de Datos

### 1. Modelo de Datos (Firestore)
- **Colección**: `realEstateTerrains`
- **Campos**:
  - `orgId`: String (Aislamiento por organización)
  - `city`: String
  - `district`: String
  - `address`: String
  - `ownerId`: String (Relación con CRM)
  - `brokers`: Array de Strings (Externos)
  - `area`: Number
  - `pricePerM2`: Number
  - `totalPrice`: Number
  - `notes`: String
  - `status`: String ('presentacion', 'negociacion', 'aprobado', 'descartado')
  - `buyerId`: String (Opcional, para el Kanban)
  - `createdAt`: Timestamp

### 2. Hooks y Estado
- Crear `useRealEstate.js` para CRUD y suscripción en tiempo real.
- Integrar `useCrm` para obtener la lista de contactos (propietarios y compradores).
