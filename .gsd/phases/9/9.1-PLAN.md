# Plan 9.1: Estado Global de Módulos (Tenant Settings)

## Objective
Implementar el estado global (`useTenant` o adaptar `useAuth`) para que la aplicación conozca los módulos activos del usuario y proteja dinámicamente la navegación del Sidebar y el acceso a las rutas.

## Context
- app/.gsd/SPEC.md
- app/.gsd/ROADMAP.md
- app/src/context/AuthContext.jsx
- app/src/layouts/client/ClientLayout.jsx
- app/src/App.jsx

## Tasks

<task type="auto">
  <name>Implementar Módulos Activos en Auth/State</name>
  <files>
    app/src/context/AuthContext.jsx
    app/src/layouts/client/ClientLayout.jsx
  </files>
  <action>
    - En el payload mock de login o Firestore para el usuario actual, asegúrate de proveer un campo `activeModules: ['crm', 'inventory']` (por defecto o cargado de la DB).
    - Actualizar `ClientLayout.jsx` (Sidebar o Topbar) para renderizar los enlaces del menú lateral sólo si la clave de ruta está dentro del array `activeModules` del usuario actual.
    - Asegurar que siempre se muestre la opción persistentemente visible para "Marketplace" y "Dashboard".
  </action>
  <verify>grep -q "activeModules" app/src/context/AuthContext.jsx || echo "Missing activeModules"</verify>
  <done>El menú de navegación sólo renderiza los enlaces permitidos según el estado activo de módulos, además del enlace a Marketplace.</done>
</task>

<task type="auto">
  <name>Protección Dinámica de Rutas</name>
  <files>
    app/src/App.jsx
    app/src/components/ModuleRoute.jsx
  </files>
  <action>
    - Crear un componente verificador `ModuleRoute.jsx` (o usar en `App.jsx`) que compruebe si el usuario tiene activado el módulo respectivo.
    - Si intenta acceder sin acceso, se le debe redirigir a `/client/marketplace`.
    - Envolver los módulos protegidos (crm, inventory, sales, finance, calendar, projects) con este verificador en el enrutamiento.
  </action>
  <verify>grep -q "ModuleRoute" app/src/App.jsx || echo "Missing ModuleRoute"</verify>
  <done>Navegar a una ruta de un módulo no activado redirigirá al Marketplace.</done>
</task>

## Success Criteria
- [ ] Sólo se visualizan módulos permitidos en el navbar.
- [ ] Un usuario no puede acceder directamente por URL a una ruta si su módulo se encuentra desactivado.
