# Plan 50.3: Control de Sesiones Activas

## Objective
Monitorizar las sesiones activas de los usuarios y permitir a los administradores invalidarlas remotamente por razones de seguridad.

## Context
- .gsd/SPEC.md
- src/context/AuthContext.jsx
- src/modules/client/team/TeamModule.jsx

## Tasks

<task type="auto">
  <name>Lógica de Cierre Remoto</name>
  <files>src/context/AuthContext.jsx</files>
  <action>
    - Modificar el listener de `onSnapshot` del documento del usuario en `AuthContext`.
    - Revisar si existe el campo `sessionRevokedAt`. Si este valor es mayor que el timestamp actual (o se actualizó recientemente), ejecutar `signOut(auth)` forzando la salida del usuario.
    - Opcional: También limpiar el sessionStorage (`sessionStorage.removeItem('pkt_user_session')`).
  </action>
  <verify>grep -A 5 "sessionRevokedAt" src/context/AuthContext.jsx</verify>
  <done>El contexto vigila el campo de revocación y expulsa al usuario activo si es necesario.</done>
</task>

<task type="auto">
  <name>Botón de Cierre de Sesión en TeamModule</name>
  <files>src/modules/client/team/TeamModule.jsx</files>
  <action>
    - Añadir una opción "Cerrar todas las sesiones" en el menú de acciones de cada usuario en la tabla de equipo.
    - Al confirmar, actualizar el documento de ese usuario en `organizations/{orgId}/users/{userId}` seteando `sessionRevokedAt: serverTimestamp()`.
  </action>
  <verify>grep -q "sessionRevokedAt" src/modules/client/team/TeamModule.jsx</verify>
  <done>El administrador puede invocar el cierre remoto de sesiones de sus colaboradores.</done>
</task>

## Success Criteria
- [ ] Botón disponible para invalidar sesiones en el módulo de Equipo.
- [ ] Actualizar la base de datos con una orden de revocación desloguea automáticamente al cliente afectado.
