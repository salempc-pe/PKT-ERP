# Plan 43.2: Lógica de Valorización y Transferencias

## Objective
Implementar el motor transaccional de inventario para calcular el Promedio Ponderado de valorización y manejar el estado de transferencias entre almacenes.

## Context
- .gsd/SPEC.md
- src/modules/client/inventory/useInventory.js

## Tasks

<task type="auto">
  <name>Motor de Transacciones y Promedio Ponderado</name>
  <files>
    - src/modules/client/inventory/useInventoryTransactions.js
    - src/modules/client/inventory/useInventory.js
  </files>
  <action>
    - Crear `useInventoryTransactions.js` con soporte para colección `organizations/{orgId}/inventory_transactions`.
    - Implementar la función `recordTransaction({ type, productId, quantity, unitCost, warehouseId, notes })`.
    - `type` puede ser 'inbound' (entrada) o 'outbound' (salida).
    - Para 'inbound', la función debe:
      1. Leer el `stock` y `averageCost` actual del producto.
      2. Calcular nuevo costo promedio: `((stock_actual * costo_promedio) + (cantidad * costo_unitario)) / (stock_actual + cantidad)`.
      3. Actualizar el documento del producto (incrementar stock total, actualizar `stockByWarehouse[warehouseId]`, setear nuevo `averageCost`).
    - Para 'outbound', simplemente descontar el stock general y el `stockByWarehouse`, usando el `averageCost` actual como costo de salida para propósitos de reporte.
  </action>
  <verify>grep_search() para confirmar que la fórmula de promedio ponderado está implementada correctamente en el hook.</verify>
  <done>Las entradas de stock recalculan matemáticamente el costo promedio de inventario del producto.</done>
</task>

<task type="auto">
  <name>Sistema de Transferencias en Tránsito</name>
  <files>
    - src/modules/client/inventory/useInventoryTransfers.js
  </files>
  <action>
    - Crear `useInventoryTransfers.js` para la colección `organizations/{orgId}/inventory_transfers`.
    - Estructura: `{ id, productId, fromWarehouseId, toWarehouseId, quantity, status, createdAt, completedAt, createdBy }`.
    - `status`: 'in_transit', 'completed', 'cancelled'.
    - Implementar `initiateTransfer`: Descuenta de `fromWarehouseId` y crea el documento en estado `in_transit`.
    - Implementar `completeTransfer`: Incrementa `toWarehouseId` y marca como `completed`.
    - Implementar `cancelTransfer`: Devuelve el stock a `fromWarehouseId` y marca `cancelled`.
  </action>
  <verify>grep_search() de los métodos initiateTransfer y completeTransfer comprobando manipulación de stock.</verify>
  <done>El stock puede viajar de un almacén a otro pasando por un estado temporal "en tránsito".</done>
</task>

## Success Criteria
- [ ] La recepción de nuevo lote de producto re-calcula su costo promedio de forma atómica.
- [ ] Se puede reservar stock local moviéndolo a un estado "en_transito" a otro almacén.
