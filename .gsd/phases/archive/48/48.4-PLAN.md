# Plan 48.4: Repositorio Legal de Documentos

## Objective
Proveer una interfaz dentro de los detalles del terreno para gestionar los documentos legales adjuntos (Partidas Registrales, Minutas, HR, PU, etc). 

## Context
- src/modules/client/realestate/components/TerrainDetailsModal.jsx
- src/utils/fileUtils.js (opcional, para conversión de Base64 si se requiere)

## Tasks

<task type="auto">
  <name>Implement Legal Documents Tab</name>
  <files>src/modules/client/realestate/components/TerrainDetailsModal.jsx</files>
  <action>
    Dentro de `TerrainDetailsModal`, en la sección "Documentos Legales":
    - Mostrar una lista de documentos existentes en el array `terrain.documents`.
    - Cada fila debe mostrar el nombre del documento, tipo y un botón para "Ver/Descargar" (si es una URL o Base64, abrir en una nueva pestaña).
    - Añadir un botón para "Subir Documento". Al hacer clic, debe solicitar un archivo mediante un `<input type="file">`.
  </action>
  <verify>Select-String -Path "src/modules/client/realestate/components/TerrainDetailsModal.jsx" -Pattern "documents" -Quiet</verify>
  <done>Se pueden listar los documentos legales asociados al terreno.</done>
</task>

<task type="auto">
  <name>Implement Document Upload Logic</name>
  <files>src/modules/client/realestate/components/TerrainDetailsModal.jsx</files>
  <action>
    Al seleccionar un archivo:
    1. Utilizar un FileReader para convertir archivos pequeños (PDFs o imágenes menores a 1MB) a Base64.
    2. En producción real esto iría a Firebase Storage, pero para cumplir con el alcance y mantener el costo en $0, guardaremos el base64 directamente en Firestore dentro del objeto `documents` (o mockearemos la carga si el tamaño es muy restrictivo).
    3. Construir el objeto de documento: `{ id: randomUUID, name: file.name, type: file.type, url: base64Data, uploadedAt: new Date().toISOString() }`.
    4. Actualizar el terreno con `updateTerrain(terrain.id, { documents: [...newDocuments] })`.
  </action>
  <verify>Select-String -Path "src/modules/client/realestate/components/TerrainDetailsModal.jsx" -Pattern "FileReader" -Quiet</verify>
  <done>Se pueden subir documentos a la base de datos (como Data URIs temporales o permanentes) y asociarlos al terreno.</done>
</task>

## Success Criteria
- [ ] La UI permite cargar documentos.
- [ ] Es posible ver/descargar los documentos previamente cargados.
