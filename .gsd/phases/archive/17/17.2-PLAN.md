# Plan 17.2: Gestión de Crédito en CRM

## Objective
Habilitar la configuración de plazos de crédito (en días) para cada cliente en el módulo CRM, lo cual servirá para el cálculo automático de vencimientos en facturación.

## Context
- app/src/modules/client/crm/useCrm.js
- app/src/modules/client/crm/CRMModule.jsx

## Tasks

<task type="auto">
  <name>Extensión del Modelo de Cliente (Crédito)</name>
  <files>app/src/modules/client/crm/useCrm.js</files>
  <action>
    - Añadir el campo `creditDays` (tipo número) al modelo de contacto/cliente.
    - Asegurar que la función `addContact` persista este nuevo campo.
    - Actualizar la data mock para incluir algunos clientes con crédito (ej: 15 o 30 días).
  </action>
  <verify>Verificar que useCrm.js maneje el campo creditDays en el estado y persistencia.</verify>
  <done>Los clientes pueden tener un valor de 'creditDays' asociado en la base de datos.</done>
</task>

<task type="auto">
  <name>Formulario CRM con Campo de Crédito</name>
  <files>app/src/modules/client/crm/CRMModule.jsx</files>
  <action>
    - Agregar un campo de entrada (input number) en el modal de "Nuevo Cliente" para definir los "Días de Crédito".
    - Actualizar el estado del formulario (`formData`) para incluir `creditDays`.
    - (Opcional) Mostrar el crédito asignado en la tabla de Base de Contactos si se desea visibilidad inmediata.
  </action>
  <verify>Verificar que el modal de nuevo cliente permita ingresar los días de crédito y los guarde.</verify>
  <done>La interfaz del CRM permite capturar los términos de crédito de cada cliente.</done>
</task>

## Success Criteria
- [ ] Los clientes pueden registrarse con un número específico de días de crédito.
- [ ] El CRM persiste y recupera correctamente el valor de `creditDays`.
- [ ] La UI permite ingresar este valor de forma intuitiva.
