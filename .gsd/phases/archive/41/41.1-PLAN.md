# Plan 41.1: Personalización de Widgets del Dashboard

## Objective
Permitir a los usuarios configurar qué métricas (tarjetas de módulos) desean ver en su dashboard principal, guardando esta preferencia en su perfil.

## Context
- .gsd/SPEC.md
- src/modules/client/dashboard/ClientDashboard.jsx
- src/context/AuthContext.jsx

## Tasks

<task type="auto">
  <name>Componente DashboardSettingsModal</name>
  <files>src/modules/client/dashboard/DashboardSettingsModal.jsx</files>
  <action>
    - Crear un nuevo componente modal para la personalización de widgets.
    - Recibirá `isOpen`, `onClose`, `activeModules` (los módulos de la suscripción de la organización) y `userPreferences` (los módulos actualmente visibles elegidos por el usuario).
    - Mostrará una lista de "switches" (toggles) para cada módulo.
    - Al guardar, utilizará `updateDoc` en `doc(db, 'users', user.id)` para actualizar el array `dashboardPreferences`.
    - Gracias al listener reactivo del `AuthContext`, el usuario se actualizará automáticamente.
  </action>
  <verify>grep -q "dashboardPreferences" src/modules/client/dashboard/DashboardSettingsModal.jsx</verify>
  <done>El componente modal existe y contiene la lógica para actualizar las preferencias en Firestore.</done>
</task>

<task type="auto">
  <name>Integración de Personalización en el Dashboard</name>
  <files>src/modules/client/dashboard/ClientDashboard.jsx</files>
  <action>
    - Importar e integrar el nuevo `DashboardSettingsModal`.
    - Añadir un botón (ej. un ícono de "Settings" o "Widgets") en la sección de encabezado del Dashboard (junto al saludo) para abrir el modal.
    - Filtrar la variable `activeModules` que se usa para renderizar las tarjetas: si `user.dashboardPreferences` existe y es un array, mostrar solo los módulos que estén en ese array. Si no existe, mostrar todos los de la suscripción por defecto.
  </action>
  <verify>grep -q "DashboardSettingsModal" src/modules/client/dashboard/ClientDashboard.jsx</verify>
  <done>El Dashboard principal muestra el botón de configuración y filtra los widgets mostrados según las preferencias del usuario.</done>
</task>

## Success Criteria
- [ ] El usuario puede abrir un modal para elegir qué tarjetas ver en el dashboard.
- [ ] Las selecciones se guardan en el perfil de Firestore.
- [ ] El dashboard oculta las tarjetas no seleccionadas de forma reactiva.
