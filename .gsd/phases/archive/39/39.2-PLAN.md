# Plan 39.2: Integraciones y Webhooks (Notificaciones)

## Objective
Preparar el módulo de Agenda para enviar datos (vía webhooks) a servicios externos como Make/Zapier para automatizar notificaciones por correo o WhatsApp, y habilitar un panel de configuración de integraciones.

## Context
- .gsd/SPEC.md
- src/modules/client/calendar/useCalendar.js
- src/modules/client/calendar/CalendarModule.jsx

## Tasks

<task type="auto">
  <name>Panel de Integraciones</name>
  <files>
    src/modules/client/calendar/CalendarModule.jsx
    src/modules/client/calendar/IntegrationsModal.jsx
  </files>
  <action>
    - Crear `IntegrationsModal.jsx` para configurar URLs de webhooks (ej. `webhookUrlCreation`, `webhookUrlReminder`).
    - Añadir un botón "Integraciones" o "Configuración" en la UI superior de `CalendarModule.jsx` que abra este modal.
    - Implementar guardado de esta configuración en el documento de la organización (`organizations/{orgId}`) en un campo `settings.calendarWebhooks`.
  </action>
  <verify>grep -q "IntegrationsModal" src/modules/client/calendar/CalendarModule.jsx</verify>
  <done>El usuario puede abrir el panel y configurar URLs de webhooks.</done>
</task>

<task type="auto">
  <name>Disparo de Notificaciones en Eventos</name>
  <files>src/modules/client/calendar/useCalendar.js</files>
  <action>
    - Modificar `useCalendar.js` para leer la configuración de los webhooks de la organización.
    - Tras crear (`addEvent`) o actualizar (`updateEvent`) exitosamente un evento, usar la API `fetch` para hacer un HTTP POST a la URL del webhook configurado, enviando el payload del evento (título, fecha, hora, descripción).
    - Usar `try/catch` para que un fallo en el webhook externo no bloquee la creación local del evento en Firebase.
  </action>
  <verify>grep -q "fetch" src/modules/client/calendar/useCalendar.js</verify>
  <done>El sistema intenta notificar silenciosamente a un webhook tras modificaciones de la agenda.</done>
</task>

## Success Criteria
- [ ] Panel de configuración de webhooks disponible en la UI.
- [ ] Creación de eventos dispara notificaciones externas (POST a webhook).
- [ ] El fallo del webhook no detiene el funcionamiento nativo de la app.
