# Summary 44.1: Refactorización de Datos (Bodega)

## Lo que se hizo
- Se refactorizó el hook `useWarehouse.js` para soportar el parámetro `selectedWarehouseId`, permitiendo filtrar stock e historial por almacén físico.
- Se actualizó la función `addMovement` para registrar el `warehouseId` en cada movimiento y calcular el `movementValue` en las salidas.
- Se implementó el nuevo hook `useMaterialSettings.js` para gestionar los umbrales de stock mínimo por material y almacén en la colección `material_settings`.
