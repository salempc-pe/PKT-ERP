# Plan 14.1: AuthContext Logic & Admin UI for Onboarding

## Objective
Implementar lógica de estados de usuario (Pending/Active) y tokens de invitación en el `AuthContext`, y actualizar la UI del Admin (`AdminClients.jsx`) para mostrar el estado y permitir copiar el "Link de Invitación".

## Context
- .gsd/SPEC.md
- src/context/AuthContext.jsx
- src/modules/admin/clients/AdminClients.jsx

## Tasks

<task type="auto">
  <name>Update AuthContext for Invitations</name>
  <files>src/context/AuthContext.jsx</files>
  <action>
    - Modificar `INITIAL_MOCK_USERS` para que cada uno posea explícitamente `status: 'active'`.
    - Modificar la función `adminCreateUser` para estampar en los nuevos usuarios `status: 'pending'`, remover la contraseña por defecto ('1234') y generar un `inviteToken` único (ej. `Math.random().toString(36).substring(2)`).
    - Hacer que `adminCreateUser` retorne `{ success: true, inviteToken }` u objeto adecuado que traiga el token.
    - Implementar una función `setupUserPassword(token, newPassword)` en `AuthContext.jsx` que ubica al usuario con el `inviteToken`, le asigne la clave `newPassword`, lo cambie a `status: 'active'`, reseteé el token (removerlo) y actualice `mockUsers`.
  </action>
  <verify>grep -q "setupUserPassword" src/context/AuthContext.jsx</verify>
  <done>AuthContext gestiona tokens de invitación y el pipeline de configuración de passwords.</done>
</task>

<task type="auto">
  <name>Admin UI - Link de Invitación</name>
  <files>src/modules/admin/clients/AdminClients.jsx</files>
  <action>
    - En el listado `organizationsWithUsers.map(u => ...)` mostrar una etiqueta (badge) con el status del usuario (Active o Pending). 
    - Al llamar `adminCreateUser(..., newUserInOrg)`, si la creación es exitosa, mostrar el link de un solo uso en la UI de inmediato para ese usuario (o en un recuadro temporal), pero sobre todo permitir **copiar** ese link.
    - A modo sencillo: mostrar en la tarjeta flotante en `AdminClients.jsx` al lado de los usuarios 'Pending' un botón "Copiar enlace de acceso" que copie la URL equivalente a `window.location.origin + '/setup-password?token=' + user.inviteToken`. Ojo que `AuthContext` necesita retornar este token en el listado (`getClientUsers`), así que asegurarse de que el token se guarda en el objeto del usuario o se crea.
    - Usar `navigator.clipboard.writeText` para el botón copiar y mostrar una notificación leve de éxito.
  </action>
  <verify>grep -q "Pending" src/modules/admin/clients/AdminClients.jsx</verify>
  <done>La interfaz UI del Admin hace fácil y visible gestionar estados y enviar links a usuarios pendientes.</done>
</task>

## Success Criteria
- [ ] Los usuarios nuevos nacen como 'Pending' y se les asigna link de invitación.
- [ ] El administrador tiene siempre una vía de copiar y proporcionar ese link.
