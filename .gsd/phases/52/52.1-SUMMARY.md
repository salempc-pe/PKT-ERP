# Summary Plan 52.1

Se actualizó el listener en tiempo real `onSnapshot` en el `AuthContext` para que detecte y propague cambios no solo en roles y permisos, sino también en las preferencias del dashboard (`dashboardPreferences`) y en los campos de perfil (`name`, `photoUrl`, `documentId`, `position`). 

Para evitar el "closure trap", se refactorizó el chequeo de diferencias dentro de la función callback del actualizador `setUser(prev => ...)`, garantizando que las comparaciones siempre se realicen con el estado React más actual sin forzar la reinicialización del suscriptor de Firebase.
