# Plan 50.2: Registro de Auditoría (Audit Log)

## Objective
Crear un sistema centralizado para registrar acciones críticas de los usuarios y exponer este historial en la interfaz para los administradores de la organización.

## Context
- .gsd/SPEC.md
- src/modules/client/team/TeamModule.jsx

## Tasks

<task type="auto">
  <name>Hook y Colección de Auditoría</name>
  <files>src/hooks/useAuditLog.js</files>
  <action>
    - Crear `useAuditLog.js` exportando la función `logAction(action, module, details)`.
    - Esta función utilizará el `orgId` y `user` actual (desde `AuthContext` si es posible o pasados como argumentos) para escribir en la subcolección `organizations/{orgId}/audit_logs`.
    - Crear el hook secundario `useGetAuditLogs(orgId)` para recuperar los logs recientes ordenados por `timestamp` descendente.
  </action>
  <verify>grep -q "audit_logs" src/hooks/useAuditLog.js</verify>
  <done>Existe una función reutilizable para registrar acciones en la base de datos.</done>
</task>

- [ ] La UI muestra los eventos cronológicamente a los administradores.
