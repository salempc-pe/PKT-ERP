# Plan 50.1: Sistema de ACL Granular

## Objective
Implementar un sistema de Control de Acceso basado en Listas (ACL) para otorgar permisos finos (Leer, Crear, Editar, Eliminar) por módulo a los colaboradores de una organización.

## Context
- .gsd/SPEC.md
- src/context/AuthContext.jsx
- src/modules/client/team/TeamModule.jsx
- src/modules/client/team/UserModal.jsx

## Tasks

<task type="auto">
  <name>Helper de Permisos en AuthContext</name>
  <files>src/context/AuthContext.jsx</files>
  <action>
    - Añadir una función `hasPermission(moduleKey, actionKey)` en el `AuthContext`.
    - La lógica debe ser: si el usuario es `admin`, devolver `true`. Si es `user`, revisar el objeto `userProfile.permissions[moduleKey][actionKey]`.
    - Si el objeto `permissions` no existe o no tiene ese módulo, devolver `false` (por defecto cerrado).
    - Exportar `hasPermission` junto con el `user`.
  </action>
  <verify>grep -A 5 "hasPermission" src/context/AuthContext.jsx</verify>
  <done>El contexto exporta una función segura para evaluar los permisos granulares del usuario activo.</done>
</task>

<task type="auto">
  <name>UI de Configuración de ACL en TeamModule</name>
  <files>
    src/modules/client/team/TeamModule.jsx
    src/modules/client/team/UserModal.jsx
  </files>
  <action>
    - En `UserModal.jsx` (o creando un nuevo componente `PermissionsModal.jsx`), permitir al Admin ver y editar una matriz de permisos para los usuarios con rol `user`.
    - Las columnas serán: Ver, Crear, Editar, Eliminar.
    - Las filas serán los módulos activos de la organización (obtenidos de `user.subscription.activeModules`).
    - Al guardar, actualizar el campo `permissions` en el documento del usuario dentro de `organizations/{orgId}/users/{userId}`.
  </action>
  <verify>grep -q "permissions" src/modules/client/team/UserModal.jsx</verify>
  <done>El administrador puede asignar permisos granulares a los miembros del equipo desde la interfaz.</done>
</task>

## Success Criteria
- [ ] La función `hasPermission` funciona y respeta el rol de administrador.
- [ ] La interfaz de gestión de equipo expone la matriz de permisos ACL para los colaboradores.
- [ ] Los permisos se persisten correctamente en Firestore.
