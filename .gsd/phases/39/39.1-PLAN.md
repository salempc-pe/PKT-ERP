# Plan 39.1: Gestión de Recursos Físicos y Conflictos

## Objective
Implementar la gestión de recursos físicos (salas, equipos) dentro del módulo de Agenda y evitar la asignación de un mismo recurso a múltiples eventos simultáneos (control de capacidad/conflictos).

## Context
- .gsd/SPEC.md
- src/modules/client/calendar/useCalendar.js
- src/modules/client/calendar/CalendarModule.jsx
- src/modules/client/calendar/EventModal.jsx

## Tasks

<task type="auto">
  <name>Actualizar Hook y Lógica de Conflictos</name>
  <files>src/modules/client/calendar/useCalendar.js</files>
  <action>
    - Modificar `useCalendar.js` para suscribirse a una nueva colección `organizations/{orgId}/resources`.
    - Crear funciones `addResource`, `updateResource`, `deleteResource`.
    - En la función `addEvent` y `updateEvent`, implementar una validación que cruce las fechas/horas (inicio y fin) del nuevo evento con los eventos existentes en el state `events`.
    - Si el evento incluye `resourceId` y se solapa en el tiempo con otro evento que usa el mismo `resourceId`, lanzar un error (ej. `throw new Error("El recurso seleccionado ya está ocupado en ese horario.");`).
    - Actualizar `EventSchema` para incluir opcionalmente `resourceId` (string).
  </action>
  <verify>grep -A 5 -B 5 "organizations.*resources" src/modules/client/calendar/useCalendar.js</verify>
  <done>El hook useCalendar recupera recursos y bloquea la creación de eventos con solapamiento de recursos.</done>
</task>

<task type="auto">
  <name>UI de Recursos y Selector en EventModal</name>
  <files>
    src/modules/client/calendar/CalendarModule.jsx
    src/modules/client/calendar/EventModal.jsx
  </files>
  <action>
    - En `CalendarModule.jsx`, pasar la lista de `resources` y las funciones de manejo a `EventModal`.
    - En `EventModal.jsx`, añadir un selector desplegable (`select`) para asignar un recurso físico al evento (`resourceId`).
    - Mostrar mensajes de error de forma elegante si ocurre un conflicto de recursos al intentar guardar.
    - Opcional: Agregar un botón o mini-modal en `CalendarModule` para "Gestionar Recursos" (añadir, editar, eliminar salas o equipos), o manejarlo como una simple lista configurable.
  </action>
  <verify>grep -C 3 "resourceId" src/modules/client/calendar/EventModal.jsx</verify>
  <done>El usuario puede asignar un recurso físico a un evento y recibe una alerta si hay conflicto.</done>
</task>

## Success Criteria
- [ ] La colección `resources` es gestionable (lectura/escritura).
- [ ] El modal de eventos permite elegir recursos.
- [ ] El sistema rechaza con un mensaje de error el solapamiento de horarios para un mismo recurso físico.
